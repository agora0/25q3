需求規劃書關聯圖

這張圖說明了文件的撰寫時機點，由BRD → MRD → PRD，要先制定未來產品藍圖(BRD)，再來進行市場調查確定這是有市場的產品(MRD)，最後要規畫整個產品的開發與細節項目(PRD)



需求規劃書給誰看?

一開始的產品藍圖需要給層級最高能夠決策公司未來的長官們看，等他們確定了後，再來要清楚目標客群與市場，最後則是要回歸到開發團隊如何去開發出產品




一. BRD 商業需求規劃書(業務需求規劃書)


適用對象: 老闆、股東、投資人、高階主管

由誰撰寫: PMO(專案管理辦公室)成員或是幕僚來撰寫


1. 說明:

當開啟一個新產品或新事業時，BRD是這項新產品或服務的規劃實現藍圖

2. 目的:

讓老闆、高階主管或是投資人認為這項產品值得投入，從而給予我們資源與投資 ，BRD需要強調產品的未來發展性、產品優勢與它能夠帶給企業什麼貢獻

  • 新產品的背景、動機與目的?

  • 產品可以幫助公司帶來哪些貢獻或利益?

  • 可以幫助公司帶來什麼優勢?

  • 怎麼運作與賺錢? 商業模式?

  • 公司需要投入多少資源?

3. 內容:

新產品可以交付出的成果與貢獻(Ex. 能夠讓公司業績提升20%)和每個開發階段所需的資源與能夠達到的成果

4. 包含的項目:

  • 必備: 產品概述、市場分析、商業模式圖、產品目標、預估損益、專案時程、風險控管與產品願景

  • 額外: 問卷調查、產品路線、定位與價值


二. MRD 市場需求規劃書


適用對象: 市場研究人員、行銷人員、營運人員、業務等

由誰撰寫: 產品負責人、產品行銷經理


1. 說明:

MRD 需要定義出目標客群(Target Audience)、產品幫助用戶解決什麼問題、營運的方向等，MRD需連結商業、市場、營運，幫助市場調查人員更好地執行分析，營運人員得到更精準的數據後，便能制定更好的市場推廣方案

2. 目的:

讓產品中的相關成員能夠清楚產品定位、未來營運方向，和各自該進行的工作

  • 確定市場需求

  • 確定目標顧客或潛在客戶

3. 內容:

  • 產品: 對產品的完整敘述，包含功能、用途、定位與願景

  • 市場: 進行市場區隔，並對目標市場進行分析，找出潛在客戶

  • 競品: 分析市場上的競爭對手與其產品

  • 營運: 確定目標客群並制定未來產品營運方向

4. 包含的項目:

必備: 市場分析、目標客群、用戶旅程、人物誌、商業模式、定位與價值、產品生命週期

額外: 產品描述、問卷調查、STP市場區隔分析、行銷4P&4C、願景


三. PRD 產品需求規格書


看公司的工作習慣，有些公司會用Spec和SA等文件替代

適用對象: 開發相關人員、測試單位等

由誰撰寫: 產品負責人


1. 說明:

PRD 清楚記錄產品的開發細節與功能，像是功能與分功能性需求，需要哪些資料表哪些欄位名稱，如何操作，產品整個運作邏輯等

2. 目的:

讓所有產品開發團隊成員、相關利害關係人清楚瞭解產品，並瞭解產品該交付什麼內容

3. 內容:

PRD 內容通常會包含產品操作介面、產品流程、功能與需求、測試、使用者體驗等

  • 概述: 目標、風險、範圍和願景

  • 功能: 清楚定義產品所有功能的列表

  • 開發架構: 產品的架構圖、透過哪些軟體、硬體等工具來開發產品

  • 資源: 列出所需的資源、工具、時間和里程碑

  • 風險管控: 釐清可能的風險和對應的解決方案

  • 結論與建議: 對於這次產品的結論與建議

4. 包含的項目:

  • 必備: 文件版控紀錄、產品結構、功能需求與細節描述(Use Case)、API、線框搞(Wireframe)、雛形(Prototype)、產品兼融性

  • 額外: 可用性測試、產品路線、數據需求、資安


結論

在讀了網路上相關文章時，發現大家會用頭、身體與心臟來比喻這三份文件，而我的理解會是BRD(頭)文件決定了整個產品策略藍圖，而MRD(身體)負責找到對的客戶與市場，最後PRD(心臟)是產品本身怎麼製作，而這三份文件從BRD → MRD → PRD 對我來說是一個收斂的過程，從公司策略→ 市場確定 → 產品開發，這一步步幫助公司完推出新產品

  • BRD: 產品的Head (Why)

  • MRD: 產品的Body (What)

  • PRD: 產品的Heart (How)
